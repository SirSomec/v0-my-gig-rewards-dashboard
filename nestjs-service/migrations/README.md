# Миграции БД (Drizzle)

При старте контейнера API автоматически выполняется скрипт `node dist/infra/db/drizzle/migrate.js`, который применяет все неприменённые миграции.

## Добавление новой миграции

1. Создайте файл `XXXX_описание.sql` (например, `0005_new_feature.sql`) в этой папке.
2. **Обязательно** добавьте запись в `meta/_journal.json` в массив `entries`:
   ```json
   {
     "idx": 4,
     "version": "7",
     "when": 1730995200000,
     "tag": "0005_new_feature",
     "breakpoints": true
   }
   ```
   - `idx` — порядковый номер (0, 1, 2, …);
   - `tag` — имя без расширения, совпадает с именем файла миграции;
   - `when` — метка времени в миллисекундах (можно взять текущее время).

Без записи в `_journal.json` Drizzle не увидит миграцию и не выполнит её.

Альтернатива: использовать `drizzle-kit generate` — тогда журнал обновится автоматически.
